Visualization Write-Up

Research Question

How does projected surface temperature change vary across the globe under different emission scenarios, and how do regional trends diverge over time? This question matters because different parts of the world face sharply different warming trajectories — the Arctic warms 2–4× faster than the global mean, and the gap between SSP2-4.5 and SSP5-8.5 compounds annually.

Design Rationale & Encoding Choices

  • Natural Earth projection (map): Chosen for its area-preserving quality at mid-latitudes and familiar appearance. Mercator was rejected because it severely exaggerates polar regions, which are the areas of greatest warming — the opposite of what we want to emphasize.
  • Diverging blue→red color scale (map): Centered at 0°C (freezing), this physically meaningful reference immediately conveys below-vs-above freezing. The palette matches the perceptual convention in climate science (blue = cold, red = hot). A sequential scale was rejected because it would hide the meaningful zero crossing.
  • Canvas raster beneath SVG borders: Climate model output is gridded data, not country-level data. Rendering each grid cell directly (rather than aggregating to countries) preserves spatial fidelity and reveals sub-national gradients — e.g., the Tibetan Plateau vs. the Ganges plain within India, or continental vs. coastal gradients across Australia.
  • Linked time series (right panel): Both SSP2-4.5 (blue) and SSP5-8.5 (red) global trajectories are shown simultaneously so the viewer can immediately perceive the divergence. Solid lines for the global mean; dashed lines for the selected country's 4 decadal values — the dash communicates that these are sparser, aggregated points rather than continuous yearly observations.
  • Decade slider rather than animation: Gives the viewer direct control over the temporal comparison, enabling deliberate juxtaposition (e.g., 2020 vs. 2080) without the cognitive load of a looping animation.
  • Crosshair + tooltip on time series: Details-on-demand — the chart is clean by default; exact temperatures surface only on hover.

Alternatives considered: a country-level choropleth (rejected: masks within-country gradients and introduces MAUP artifacts); a difference map showing 2080−2020 anomaly (considered as a mode — the decade slider achieves the same insight through interaction); animated auto-play (too passive — the slider promotes active exploration).

Data Transformations

The CMIP6 dataset was subsetted to Near-Surface Air Temperature (tas) for SSP2-4.5 and SSP5-8.5 scenarios, 2015–2100. The spatial file contains decadal means on a ~1.25° × 0.94° global grid; multiple ensemble members per cell are averaged to reduce model noise. The timeseries file contains global area-weighted annual means. No data values are excluded; the full temperature range (−52°C to +40°C) is preserved and clearly labeled.

Development Process

This is an individual submission for DSC 106. Approximate time breakdown:

  • Data acquisition and Python preprocessing (xarray, pandas, Google Colab): ~3 hrs
  • Exploratory analysis in matplotlib (6 static figures): ~2 hrs
  • D3.js map: projection, canvas raster, SVG border overlay, coordinate transforms: ~7 hrs
  • Time series chart: axes, global lines, crosshair, regional overlay, dynamic Y-scale: ~4 hrs
  • Controls, cross-panel linking, tooltips, styling: ~3 hrs
  • Write-up: ~1 hr

The most time-consuming aspect was the canvas raster rendering — specifically converting the climate model's 0–360° longitude convention to the −180–180° convention expected by D3's projections, and achieving smooth performance with 55,000+ grid cells per frame by using pre-computed projected positions and a pixel-level color LUT.
